Waterkeeper

Project Type: A project of Fifty & Fifty creative agency

Discipline: Fullstack Web Developer

Timeline: March - June 2019

Tools: Sublime Text, Adobe XD

Skills:Web development, UX Design, Strategy

Content

The waterkeeper alliance is an organization that has set it’s goal on ensuring every community worldwide has drinkable, fishable, swimmable water.

They are the largest and fastest growing nonprofit solely focused on clean water. They preserve and protect water by connecting local Waterkeeper groups worldwide.

As the senior web engineer on this project, I collaborated with strategy and design department to create sitemaps, site mock ups, userflows, and user interface design.

This case study goes over the website rebuild for waterkeeper.org. Waterkeeper alliance had the following goals for this website rebuild:

  • simplify the lookup experience for Waterkeepers in your area.
  • Show areas protected by Waterkeepers.
  • Showcase “Waterkeeper” heroes to inspire people.
  • Increase donations.

Requirements

The previous waterkeeper website was centered around the color blue to emphasize their focus on water. The waterkeeper team wanted a full redesign on the website.

The waterkeeper team wanted the new site to be very visual. They wanted to leverage large images that demonstrated clean water, and also show how pollution can affect water.

The executive director wanted a map as a visually engaging way to find out more about waterkeepers and what areas are being actively maintained.

Needfinding

We wanted to streamline the process of a user finding waterkeepers in their area. The previous website brought the user to a large list of all the waterkeepers.

This list was categories by country and was searchable but lacked a visual representation of how much affect local waterkeepers are making.

Our team wanted to find a way to show the global scale of what waterkeeper alliance is doing but bring that to a relatable local level.

Focusing on the global scale and narrowing down to the country and then to the local waterkeeper.

Design Review

The waterkeeper jurisdictions are broken into 11 different regions around the world. We needed to think of a color palette that could be used as a clear delineation of these region jurisdictions.

The user flow for navigating waterkeeper jurisdictions had the map as a central navigation piece between all steps.

Taking the opportunity of the region level to highlight stats of that region. Until the user makes their way to the local jurisdiction where they can find a bio and contact information to get involved.

Execution

I build the frontend of the site from scratch as I did not want to introduce unnecessary code that comes along with most frameworks. Their data of all the map lived in ESRI so I used ESRI’s API of ArcGIS to generate the map layer and populate it with the regions and jurisdictions.

The individual waterkeeper’s information was stored in salesforce, so I leveraged salesforce API to pull that data and populate the waterkeeper detail page.

I didn’t want the page speed to be slowed down each time that someone wanted to lookup a waterkeeper. So I built a cache mechanism to store that data from salesforce into a json array and serve the website from the static json file.

Since the updated content for waterkeepers was not an immediate need, I built the cache to grab new data on a weekly basis. This improved the page speed for the user while still providing accurate content from salesforce.

Conclusion

This was a fun visually satisfying project. Leveraging the API’s of waterkeeper alliance current data sources will allow their site to be current and dynamic for the foreseeable future.

The collaboration and team work from strategy, design and development allowed this project to be completed on time and under budget. It was a team effort that I was glad to be a part of.
